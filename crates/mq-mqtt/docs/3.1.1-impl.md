# MQTT 3.1.1 Gap Analysis & Implementation Plan

## Executive Summary

The bisque-mq-mqtt crate implements a comprehensive MQTT server supporting both 3.1.1 and 5.0 protocols. All 14 MQTT 3.1.1 control packet types are implemented with full codec support. The implementation includes QoS 0/1/2, session persistence, will messages, retained messages, topic wildcards, keep-alive, and WebSocket/TLS transports.

This document cross-references every normative MUST requirement from the MQTT 3.1.1 specification against the current implementation, identifies gaps, and provides an implementation plan.

## 2. Gap Summary

All originally identified gaps have been resolved. Below is the disposition of each:

### GAP-1: Second CONNECT packet handling [MQTT-3.1.0-2] — **RESOLVED**
Already implemented in `server.rs:1292-1314`. Second CONNECT sends DISCONNECT and closes.

### GAP-2: Offline message queuing for persistent sessions [MQTT-3.1.2-5] — **RESOLVED**
Persistent sessions keep subscriptions alive in bisque-mq. Messages queue in the subscription's queue during offline periods and drain on reconnect.

### GAP-3: Password without username validation [MQTT-3.1.2-22] — **RESOLVED**
Already implemented in `codec.rs:975-978`. V3.1.1 rejects password without username. V5 allows it per spec.

### GAP-4: Empty ClientId + CleanSession=0 rejection [MQTT-3.1.3-8] — **RESOLVED**
Already implemented in `session.rs:586-604`. Returns CONNACK 0x02 (IdentifierRejected).

### GAP-5: Overlapping subscription QoS delivery [MQTT-3.3.5-1] — **RESOLVED**
The spec says server MUST deliver at max QoS and MAY deliver further copies. Current one-copy-per-subscription behavior is explicitly allowed.

### GAP-6: Re-send retained messages on subscription replacement [MQTT-3.8.4-3] — **RESOLVED**
V3.1.1 uses retain_handling=0 (default), so retained messages are always re-sent on re-subscribe, including replacements.

### GAP-7: Session reconnect retransmission [MQTT-4.4.0-1] — **RESOLVED**
**Fixed:** `should_persist()` and `extract_session_state()` in `session_store.rs` were corrected for V3.1.1:
- V3.1.1 `clean_session=false` now always persists (was broken by checking `session_expiry_interval > 0`)
- V3.1.1 sessions use `0xFFFFFFFF` (never expire) since V3.1.1 has no session expiry concept
- `pending_retransmits()` generates PUBREL retransmissions for QoS 2 in-flight messages
- Session store persists pending QoS 1/2 packet IDs for retransmission on reconnect

### GAP-8: $-topic wildcard filtering [MQTT-4.7.2-1] — **RESOLVED**
Already implemented in `server.rs:1661` (topic matching) and `server.rs:736-752` (delivery loop filtering).
